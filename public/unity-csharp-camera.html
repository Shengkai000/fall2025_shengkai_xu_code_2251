Camera Follow

Unity C# Camera

Smooth camera follow with clamped bounds

Make the camera follow a target (like a player) with smooth damping and optional horizontal/vertical limits, ideal for platformers and top-down games.

Base snippet


// Smooth camera follow with optional bounds.

using UnityEngine;

public class CameraFollow : MonoBehaviour
{
    public Transform target;
    public float smoothTime = 0.2f;

    [Header("Optional bounds")]
    public bool useBounds = false;
    public Vector2 minBounds;
    public Vector2 maxBounds;

    private Vector3 velocity = Vector3.zero;

    void LateUpdate()
    {
        if (target == null) return;

        Vector3 desired = new Vector3(target.position.x, target.position.y, transform.position.z);

        if (useBounds)
        {
            desired.x = Mathf.Clamp(desired.x, minBounds.x, maxBounds.x);
            desired.y = Mathf.Clamp(desired.y, minBounds.y, maxBounds.y);
        }

        transform.position = Vector3.SmoothDamp(transform.position, desired, ref velocity, smoothTime);
    }
}

Integration Recipe

  1. Attach CameraFollow to your main Camera GameObject.
  2. In the Inspector, drag your player (or target) Transform into the Target field.
  3. Play with Smooth Time to control how snappy or floaty the camera feels.
  4. To limit camera movement, enable Use Bounds and set Min Bounds / Max Bounds based on your level size.

Common Errors

  • Camera doesn't move – Make sure the Target field is set and that your player is actually moving.
  • Camera shakes or jitters – Check if you also move the camera from other scripts; only one script should control the camera's transform.
  • Camera stops unexpectedly – If bounds are enabled, verify the min/max values; your target might be clamped at the edges.
